基础篇

列表与元组

概述

列表是动态的，长度大小不固定，可以随意的增加、删减或者改变元素。

元组是静态的，长度大小固定，无法增加删减或者改变。
如果想对元组进行改变，只能重新开辟一块内存空间，创建新的元组。

都支持负数索引、切片操作、可以随意嵌套、可以通过list()和tuple()函数相互转换。

常用的内置函数

其中reverse()，sort()分别表示原地倒转列表和排序（元组中没有这两个函数）
reversed()返回倒转后的迭代器，sorted()返回排好序的新列表

存储方式的差异

列表是动态的需要储存指针，空列表需要提前分配空间，以及时分配额外空间。

元组的储存空间固定，更加轻量。

内部实现方式都是array

性能的差异

元组的性能速度要优与列表。

python会对静态数据进行资源缓存，通常由于垃圾回收机制的存在，如果一些变量不会用，python就会回收，但是对静态变量如占用空间不大的元组，会暂时缓存，下次用时直接从缓存中取，提高速度。

使用场景

如果储存的数据和数量不变，用元组
如果储存的数据和数量可变，用列表

indices/entries

插入操作的时候会将新的entries的哈希值和键与旧的进行比较
如果有哈希值相同键不同就是哈希冲突，会继续线性寻找直到找到空位

旧哈希表当内存空间小于1/3的时候，为了避免哈希冲突就会扩容，重新分配更大的内存，所以旧哈希表会越来越稀疏。

函数

函数式编程的思想，代码中每一块都是不可变的(immutable)，都用纯函数的形式组成，纯函数本身相互独立，互不影响，对于相同的输入总有相同的输出。

1.函数可以返回调用结果（return或yield），也可以不返回
2.函数内部不可以随意改变全局变量的值，加上global之后可以修改
3.嵌套函数内部修改外部的加nonlocal

函数的嵌套：
可以保证内部函数的隐私
合理的使用函数嵌套，能够提高程序的运行效率

闭包
